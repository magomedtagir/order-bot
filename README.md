# Order Bot

Telegram-бот для парсинга и управления заказами из группового чата.

## Требования

- Python 3.11+
- pip

## Установка

```bash
cd order-bot
pip install -r requirements.txt
```

## Конфигурация

Скопируйте `.env.example` в `.env` и заполните переменные:

```bash
cp .env.example .env
```

| Переменная      | Описание                                              | По умолчанию                          |
|-----------------|-------------------------------------------------------|---------------------------------------|
| `BOT_TOKEN`     | Токен бота от [@BotFather](https://t.me/BotFather)   | **обязательно**                       |
| `ADMIN_IDS`     | Telegram user_id администраторов через запятую        | (все пользователи)                    |
| `DATABASE_URL`  | URL базы данных SQLAlchemy (async)                    | `sqlite+aiosqlite:///orders.db`       |
| `LOG_LEVEL`     | Уровень логирования                                   | `INFO`                                |

## Инициализация базы данных

```bash
python init_db.py
```

Создаёт таблицы и загружает начальный справочник синонимов из `data/synonyms.json`.

## Запуск

```bash
python main.py
```

## Формат заказа

Бот распознаёт сообщения в следующем формате:

```
🔴🔴🔴🔴 Тропики :
Маковая начинка 15 ведер
Сухой мак 2 мешка
Вареная сгущенка 10 вед
```

- **Первая строка** — название клиента (эмодзи, двоеточия и пробелы игнорируются)
- **Каждая последующая строка** — позиция заказа: `Название количество единица`

## Команды бота

### Просмотр заказов

| Команда       | Описание                               |
|---------------|----------------------------------------|
| `/orders`     | Последние 10 заказов со статусами      |
| `/order 5`    | Детали заказа #5 (позиции, история)    |
| `/unknown`    | Позиции, которые не удалось нормализовать |

### Управление статусами

```
/status <номер> <статус>
```

| Ключ          | Статус                        |
|---------------|-------------------------------|
| `queued`      | 🟡 В очереди                  |
| `processing`  | 🔵 Принят в обработку         |
| `delivery`    | 🟠 Запланирована доставка      |
| `delivered`   | 🟢 Доставлен                  |

Пример: `/status 3 processing`

При смене статуса бот отправляет уведомление в чат, откуда пришёл заказ.

### Справочник синонимов

| Команда                                    | Описание                        |
|--------------------------------------------|---------------------------------|
| `/synonyms`                                | Показать все синонимы           |
| `/add_synonym "вар сгущ" "Вареная сгущенка"` | Добавить/обновить синоним     |

---

## Обработка редактирований

- Если статус заказа `queued` — заказ перепарсируется и обновляется.
- Если статус изменился — бот уведомляет, что изменения не применены.
